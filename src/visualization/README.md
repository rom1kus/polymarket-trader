# Trading Visualization Dashboard

Interactive web-based dashboard for analyzing Polymarket trading history.

## Quick Start

```bash
npm run visualize
```

This will:
1. Scan `data/` directory for trading session files (`fills-*.json`)
2. Generate a manifest of all available sessions
3. Start a local web server at `http://localhost:3000`
4. Open the dashboard in your default browser

## Technical Details

### Architecture
- **Zero dependencies** - Pure HTML/JS with Chart.js from CDN
- **No build process** - Just run the npm script
- **Client-side processing** - All data is loaded and processed in the browser
- **Auto-generated manifest** - Sessions are discovered automatically on launch

### Files
- `index.html` - Main dashboard UI
- `app.js` - Chart rendering and data processing logic
- `updateManifest.js` - Manifest generator (runs before server starts)
- `manifest.json` - Auto-generated session index

### Data Source
The dashboard reads trading history from `data/fills-*.json` files, which are automatically generated by the market maker strategy when fills occur.

## Workflow

## Troubleshooting

**"No sessions found"**
- Make sure you have JSON files in the `data/` directory
- Files must be named `fills-*.json`

**Charts not loading**
- Check browser console for errors
- Ensure you're accessing via the npm script (not opening the HTML file directly)
- Verify the manifest.json was generated correctly

**Port 3000 already in use**
- Stop other services using port 3000
- Or edit `package.json` to use a different port in the `visualize` script
